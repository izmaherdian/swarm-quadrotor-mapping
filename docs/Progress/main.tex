\section{Pendahuluan}

Proyek ini mengembangkan sistem simulasi swarm quadrotor untuk pemetaan geodesi area non-konveks secara otonom. Sistem menggunakan arsitektur tiga lapis kecerdasan --- \textbf{High-Level} (koordinasi geometris), \textbf{Mid-Level} (collision avoidance), dan \textbf{Low-Level} (robust control) --- di atas ROS 2 Lyrical Luth dan Gazebo Harmonic.

Tujuan akhir adalah publikasi di Engineering Physics International Conference (EPIC). Hingga saat ini seluruh fase dasar (0--3) telah selesai. Laporan ini menjelaskan metodologi setiap layer secara sistematis.

\subsection{Low-Level: Robust Control}

Lapisan ini mengendalikan dinamika quadrotor 6-DOF. Parameter fisik: $m=1.0$ kg, $I_x=I_y=8.1\times10^{-3}$, $I_z=14.2\times10^{-3}$ kg$\cdot$m$^2$, $l=0.24$ m. Sistem didekomposisi jadi 4 subsistem terdekopel: Longitudinal ($X\leftrightarrow\theta$), Lateral ($Y\leftrightarrow\phi$), Altitude ($Z$), dan Yaw ($\psi$). Masing-masing menggunakan PID kaskade (outer loop posisi $\to$ sudut, inner loop sudut $\to$ torsi).

    \label{fig:lowlevel}

\textbf{Poin-poin:}
\begin{itemize}[nosep]
    \item \textbf{PID-LQR:} Gain PID diturunkan dari LQR optimal control. Sistem diperluas (\textit{augmented}) dengan integrator, lalu diselesaikan via \textit{Continuous Algebraic Riccati Equation} (CARE):
          \begin{equation}
              A_{\text{aug}}^T P + P A_{\text{aug}} + Q_{\text{aug}} - P B_{\text{aug}} R_{\text{aug}}^{-1} B_{\text{aug}}^T P = 0
          \end{equation}
          Gain state-feedback $K_a = R_{\text{aug}}^{-1} B_{\text{aug}}^T P$ dipetakan ke $(K_p, K_i, K_d)$ melalui matriks $\Gamma$ dan pseudoinverse: $\hat{K} = \Gamma^{\dagger} K_a^T$.

    \item \textbf{PID-Hinf:} Menggunakan kontrol robust dengan jalur disturbance $B_w$. Matriks Riccati dimodifikasi:
          \begin{equation}
              A_{\text{aug}}^T P + P A_{\text{aug}} + Q_{\text{aug}} - P \tilde{B} \tilde{R}^{-1} \tilde{B}^T P = 0,
              \quad \tilde{R} = \begin{bmatrix} R & 0 \\ 0 & -\gamma^2 I \end{bmatrix}
          \end{equation}
          Parameter $\gamma$ mengontrol robustness ($\gamma_{\text{outer}}=80$, $\gamma_{\text{inner}}=64$).

    \item Implementasi ROS 2 node (\texttt{pid\_lqr\_node.py}, \texttt{pid\_hinf\_node.py}) membaca odometri Gazebo, mengirim RPM ke motor.
    \item Model wind turbulence Dryden (Gauss-Markov orde-1) untuk pengujian ketahanan angin.
\end{itemize}

    \label{fig:midlevel}
\end{figure}

\textbf{Poin-poin:}
\begin{itemize}[nosep]
    \item \textbf{ORCA core:} Untuk setiap pasangan drone $(i,j)$, hitung \textit{Velocity Obstacle} (VO) dalam time horizon $\tau$, konversi ke ORCA half-plane $n_{ij} \cdot v' \geq b_{ij}$, lalu selesaikan \textit{Linear Programming} 2D:
          \begin{equation}
              \min_{v'} \|v' - v_{\text{pref}}\|^2 \quad \text{s.t.} \quad n_{ij} \cdot v' \geq b_{ij},\; \|v'\| \leq v_{\max}
          \end{equation}

    \item \textbf{Resiprokal:} Weight 0.5 antar drone (saling menghindar), weight 1.0 rintangan statis.
    \item \textbf{Mekanisme bantu:} repulsion field (inverse-square $<2$m), tangential gliding, COLREGs bias (turn-right), lane-keeping, slew-rate limiter (1.5 m/s$^2$), lookahead projection (0.75 detik).
    \item \textbf{DRL sempat dicoba:} PPO dilatih untuk obstacle avoidance, berhasil untuk \textbf{satu drone}. Untuk swarm multi-agent belum konvergen, sehingga beralih ke ORCA deterministik.
    \item Implementasi: \texttt{collision\_avoidance\_node.py} (623 baris).
\end{itemize}

    \label{fig:highlevel}
\end{figure}

\textbf{Poin-poin:}
\begin{itemize}[nosep]
    \item \textbf{Voronoi Partition:} Membagi area $\Omega$ menjadi sel-sel $V_i$ untuk setiap drone menggunakan Sutherland-Hodgman polygon clipping dengan bisector tegak lurus:
          \begin{equation}
              V_i = \{ \mathbf{p} \in \Omega \mid \|\mathbf{p} - \mathbf{p}_i\| \leq \|\mathbf{p} - \mathbf{p}_j\|,\; \forall j \neq i \}
          \end{equation}

    \item \textbf{Lloyd Relaxation:} 12 iterasi memindahkan titik seed drone ke centroid geometris selnya untuk menghasilkan \textit{Centroidal Voronoi Tessellation} (CVT) area proporsional:
          \begin{equation}
              \mathbf{c}_i^{(k+1)} = \frac{1}{|V_i^{(k)}|} \iint_{V_i^{(k)}} \mathbf{p} \; dA
          \end{equation}

    \item \textbf{Boustrophedon Path:} Jalur zigzag horizontal (sweep spacing 0.65m, margin 0.35m). Scanline intersection dengan poligon pada interval tetap, segmen dihubungkan bolak-balik:
          \begin{equation}
              \mathcal{W}_i = \bigcup_{y \in \mathcal{Y}_i} \text{scanline}(V_i, y)
          \end{equation}

    \item \textbf{FT-CC (Fault-Tolerant Cooperative Coverage):}
          \begin{itemize}[nosep]
              \item Deteksi kegagalan via heartbeat P2P (timeout ROS 2 DDS).
              \item Sel drone mati $k \in \mathcal{D}$ dilelehkan: $V_{\text{merged}} = \bigcup_{k \in \mathcal{D}} V_k$.
              \item Jalur Boustrophedon baru untuk area gabungan, redistribusi ke maksimal 3 drone helper.
              \item \textit{Parallel Spatial Alignment} (nol path crossing) dan \textit{Orthogonal Boundary Entry}.
          \end{itemize}
    \item Implementasi \texttt{simulator\_kinematics.py} (1.127 baris) dengan GUI keyboard interaktif (1-7: toggle drone, L: layout, B: boundary).
    \item Integrasi penuh dengan Mid+Low Level masih dalam pengerjaan (Fase 4).
\end{itemize}

\subsection{Catatan DRL}

Pendekatan Deep Reinforcement Learning (PPO) sempat diimplementasikan dan dilatih untuk obstacle avoidance. Model berhasil konvergen untuk \textbf{single agent} (1 drone dapat menavigasi ke target sambil menghindari rintangan). Namun untuk \textbf{multi-agent swarm}, pelatihan membutuhkan waktu sangat lama dan belum menunjukkan konvergensi yang memadai. Oleh karena itu, mid-level collision avoidance menggunakan ORCA deterministik yang sudah teruji untuk agen jamak. Model PPO yang sudah dilatih (format .zip dan .onnx) tetap tersimpan dan dapat digunakan kembali jika pendekatan pelatihan ditingkatkan (misalnya dengan \textit{curriculum learning} atau \textit{centralized training decentralized execution}).
